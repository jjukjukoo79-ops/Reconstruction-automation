# 정비사업 실무 자동화 — 활용 가이드

"무엇을, 어디서, 어떻게" 쓰는지 한눈에 정리한 문서입니다.
시작점은 항상 **`정비사업_허브.html`** (바로가기 허브)입니다.

---

## 1. 전체 그림 — 3개 도구 + 1개 작업실

| 구성 | 정체 | 어디서 쓰나 |
|------|------|------------|
| 📂 **정비사업 실무 에이전트** | claude.ai **프로젝트**(v2 지침) | 양식 파악·질문·서류 초안 — PC·휴대폰 |
| 🧮 **분담금 계산 스킬** | 비례율·분담금 정확 계산 | 프로젝트 안에서 "분담금 계산" |
| 📝 **동의서 징구 시스템** | 단독 실행 HTML | PC 브라우저에서 더블클릭 |
| 🛠 **Claude Code** | 도구 제작·수정·저장 | 이 PC (작업실) |

> 핵심: **질문·파악 = 프로젝트 / 도구 실행 = HTML / 제작·수정 = Claude Code**.
> 허브는 이 셋을 잇는 **링크판**일 뿐, 질문을 받는 곳이 아닙니다.

---

## 2. 어디서 무엇을 (판단 규칙)

- **"이게 무슨 단계지? 어떻게 대응? 안내문 줘"** → 📂 **프로젝트** (양식 사진/PDF 올리고 질문)
- **"분담금·비례율 정확히 계산"** → 📂 프로젝트에서 "분담금 계산"
- **"동의서/위임장 받고 동의율 집계"** → 📝 **동의서 징구 시스템**
- **"도구 새로 만들거나 고쳐줘 / 저장해줘"** → 🛠 **Claude Code(여기)**

---

## 3. 실전 흐름 예시

1. **감정평가 통보서 받음** → 📂 프로젝트에 사진 업로드 → "4단계(관리처분), 분양신청·분담금 안내가 핵심" 진단
2. **분담금 궁금** → 같은 프로젝트에서 "분담금 계산" → 수치 입력 → 비례율·분담금 표 산출
3. **동의서 모으기** → 📝 동의서 징구 시스템 → 전자/서면/스캔 등록 → 도정법 제35조 동의율 자동판정
4. **도구 변경** → 🛠 Claude Code에 요청 → 수정 후 GitHub 저장소에 보관

---

## 4. 처음 한 번 — 프로젝트 만들기 (claude.ai)

1. claude.ai → **Projects → + Create project** → 이름: `정비사업 실무 에이전트`
2. **지침(custom instructions)** 칸에 `프로젝트_지침_v2.md` 내용 통째로 붙여넣기
3. (선택) 도정법 조문·자주 쓰는 양식을 **지식 파일**로 업로드
4. (선택) 분담금 계산 스킬(`skills/redevelopment-bunbamgeum`)을 프로젝트에 추가
5. 만든 프로젝트 **주소(URL)** 를 허브의 **⚙ 프로젝트 링크 설정**에 저장 → 허브 버튼이 바로 연결됨

---

## 5. 빠르게 파악하는 요령

- **사진 한 장이면 충분** — 현장에서 양식을 찍어 프로젝트에 올리면 단계가 바로 진단됩니다.
- **배경 설명 생략 가능** — 프로젝트엔 페르소나·법조문이 이미 들어있어 매번 설명할 필요가 없습니다.
- **계산은 숫자만** — "총수입 1200억, 총사업비 700억, 종전자산 450억, 내 종전자산 4억, 분양가 6억"처럼 값만 주면 됩니다.
- **휴대폰에서도 동일** — claude.ai는 폰 웹/앱에서도 같은 프로젝트로 접속됩니다.

---

## 6. 데이터·안전

- 동의서 징구 시스템 데이터는 **이 PC 브라우저에만** 저장됩니다 → **정기적으로 JSON 백업**.
- 도구·지침의 최신본은 GitHub 저장소에 보관됩니다 → 다른 PC에서도 내려받아 사용 가능.
- 모든 산출물은 **실무 참고용 초안**이며, 최종본은 전문가 검토 후 사용하세요.
